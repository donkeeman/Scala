# Python 학습 노트 정리 스킬

## 이 스킬의 역할

사용자가 Python을 학습하면서:
1. **코드에 작성한 주석** 읽기
2. **대화 내용 분석** (사용자의 질문 + Claude의 답변)
3. **핵심 개념 추출** (사용자가 실제로 이해한 내용)
4. `docs/TIL.md`에 **날짜별로 체계적 정리**

## 사용 방법

사용자가 이렇게 요청하면 실행:
- "TIL 정리해줘"
- "오늘 배운 거 마크다운으로 정리"
- "학습 내용 정리"
- "배운 거 docs/TIL.md에 써줘"

## 스킬 실행 단계

### 1단계: 대화 내용 분석
최근 대화에서:
- 사용자의 **질문**을 찾기 (예: "__name__이 뭐야?", "typer가 뭔데?")
- Claude의 **답변**에서 핵심 개념 추출
- 사용자가 **이해했다는 신호** 찾기 (예: "오 이해했어", "됐다", "알겠어")

**중요**: 단순히 질문/답변만 기록하지 말고, **사용자가 최종적으로 이해한 내용**을 정리할 것

### 2단계: 코드 주석 읽기
프로젝트의 모든 Python 파일(`.py`)에서:
- 일반 주석 (`# 주석 내용`)
- 학습 관련 주석 특히 찾기:
  - `# LEARN:`
  - `# TIL:`
  - `# NOTE:`
  - 여러 줄 설명 주석

### 3단계: 내용 종합 및 분류
배운 내용을 **주제별로 분류**:
- **Python 기본 문법** (if문, 함수, 모듈 등)
- **Typer / CLI** (CLI 도구 관련)
- **API / HTTP** (LLM API 호출 등)
- **Git / 도구** (개발 도구 사용법)
- **기타**

### 4단계: docs/TIL.md 작성
#### 파일이 없으면 새로 생성:
```markdown
# Today I Learned

## YYYY-MM-DD

### Python 기본 문법
- 내용 1
- 내용 2

### Typer / CLI
- 내용 1
```

#### 파일이 있으면 날짜 섹션 추가:
기존 내용 유지하고 맨 위에 오늘 날짜 섹션 추가

## 작성 스타일 가이드

### ✅ 좋은 정리 예시
```markdown
## 2025-12-31

### Python 기본 문법
- `if __name__ == "__main__"`: 직접 실행할 때만 True, import되면 False
  - 테스트 코드 작성할 때 유용
  - TypeScript의 `require.main === module`과 비슷
- `==` vs `is`: 값 비교 vs 객체 비교
  - `===`는 Python에 없음

### 가상환경
- `python3 src/cli.py` ❌ (시스템 Python 사용, 패키지 없음)
- `uv run python src/cli.py` ✅ (가상환경 사용, 패키지 있음)
```

### ❌ 나쁜 정리 예시
```markdown
- 사용자가 __name__에 대해 질문함
- Claude가 설명함
- 사용자가 이해함
```

## 작성 원칙

1. **사용자 중심**: 사용자가 코드 주석으로 정리한 내용을 최우선으로
2. **극도로 간결하게**: 핵심만, 한 줄로 요약, 장황한 설명 금지
3. **짧은 예시**: 간단한 코드나 명령어만 (긴 설명 X)
4. **비교 활용**: TypeScript/JavaScript와 비교 (사용자가 프론트엔드 개발자)
5. **이모지 최소화**: 코드/명령어에만 ✅❌ 정도만

**중요**: Claude의 긴 설명보다는 사용자가 스스로 정리한 내용이 더 가치있음

## 주의사항

- **중복 방지**: 이미 TIL에 있는 내용은 추가하지 말 것
- **날짜 확인**: 오늘 날짜 섹션이 이미 있으면 거기에 추가
- **사용자 확인**: 정리 후 "어떤 내용을 TIL에 추가했는지" 간단히 요약해서 알려줄 것

## 예시 시나리오

### 사용자 요청
"TIL 정리해줘"

### 스킬 실행 과정
1. **대화 분석**:
   - 질문: "__name__ 검사가 뭐야?"
   - 이해: "직접 실행 vs import 구분"

2. **주석 읽기**:
   - `src/cli.py:6`: `# 직접 실행 시에는 __name__이 __main__`

3. **TIL 작성**:
```markdown
## 2025-12-31

### Python 기본 문법
- `if __name__ == "__main__"`: 파일 직접 실행 vs import 구분
  - 직접 실행: `__name__ == "__main__"`
  - import: `__name__ == "파일경로"`
  - 테스트 코드 작성 시 유용
```

4. **사용자에게 보고**:
"TIL에 다음 내용을 추가했습니다:
- Python 기본 문법: `__name__` 검사, 가상환경 사용법
- Typer: 함수명이 main일 필요 없음"
