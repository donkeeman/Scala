# Session Summary Skill

다른 컴퓨터의 Claude에서 대화를 이어갈 수 있도록 세션 내역을 정리하는 스킬.

## Trigger

사용자가 다음과 같이 요청할 때:
- "다른 클로드에서 이어갈 수 있도록 지금까지의 대화 내역과 진행 상황 요약해줘"
- "세션 저장해줘"
- "진행 상황 정리해줘"
- "session summary"
- "save session"

## Task

1. **대화 전체를 분석**하여 다음 정보를 추출:
   - 프로젝트 개요와 목표
   - 시간 순서대로 진행한 작업들
   - 생성/수정된 파일 목록과 주요 변경사항
   - 중요한 기술적 결정사항 (왜 이 방식을 선택했는지)
   - 해결한 문제들과 오류
   - 현재 상태 (완료/진행중/대기)
   - 다음 단계
   - 중요한 설정 (환경 변수, 패키지, 명령어 등)

2. **`docs/SESSION.md`** 파일에 다음 구조로 저장:

```markdown
# Session Summary - [날짜]

## 프로젝트 개요
- 목표: ...
- 기술 스택: ...
- 현재 단계: ...

## 진행 타임라인

### [작업 제목 1]
- 일시: [대략적인 시간]
- 내용: [무엇을 했는지]
- 결과: [생성/수정된 파일, 결정사항]

### [작업 제목 2]
...

## 생성/수정된 파일

### `파일경로`
- 목적: ...
- 주요 내용: ...
- 관련 작업: ...

## 기술적 결정사항

- **[결정 1]**: [왜 이렇게 했는지]
- **[결정 2]**: ...

## 해결한 문제

### [문제 1]
- 증상: ...
- 원인: ...
- 해결: ...

## 현재 상태

**완료:**
- [x] ...

**진행 중:**
- [ ] ...

**다음 단계:**
- [ ] ...

## 중요 설정/명령어

- 환경 변수: ...
- 패키지: ...
- 실행 명령어: ...

## 학습 내용/메모

- ...
```

3. **작성 원칙**:
   - 간결하고 명확하게
   - 다른 사람(또는 다른 Claude)이 읽고 바로 이해할 수 있도록
   - 파일 경로는 절대 경로가 아닌 프로젝트 루트 기준 상대 경로
   - 중요한 코드 스니펫은 포함하되, 전체 코드는 피함
   - 타임라인은 논리적 단위로 묶기 (너무 세분화하지 않음)

4. **저장 후**:
   - 사용자에게 "docs/SESSION.md에 세션 요약을 저장했습니다" 알림
   - 간단한 요약 출력 (프로젝트 개요, 완료된 작업 수, 다음 단계)

## Output Location

`docs/SESSION.md` (기존 파일이 있으면 덮어쓰기)

## Notes

- 이 스킬은 대화가 길어졌을 때, 또는 다른 환경으로 전환할 때 사용
- TIL.md와는 다름: TIL은 학습 내용 중심, SESSION은 작업 진행 상황 중심
- 민감한 정보(API 키 등)는 절대 포함하지 않음
